DragonbornPresence SE 2.4.0

Настройка Discord Rich Presence

Большие изображения локаций, маленькие иконки действий, приоритеты событий и диагностика assets.

Русское практическое руководство для конфигурации DragonbornPresence.json
Discord Game SDK — без OAuth и дополнительных окон авторизации

1. Быстрый старт

  1. Открой Discord Developer Portal и выбери приложение, чей ID указан в конфиге.
  2. В разделе Rich Presence → Art Assets загрузи квадратные PNG.
  3. Назначь каждому изображению уникальный Asset key: например, skyrim, combat, inventory.
  4. Открой файл Data\SKSE\Plugins\DragonbornPresence.json.
  5. Укажи Asset keys в секциях large_images, small_images и location_images.
  6. Полностью перезапусти Skyrim. Пересборка DLL после изменения JSON не требуется.
Главное правило: имя PNG может быть любым, но Asset key в Discord Portal должен точно совпадать со строкой в JSON. Регистр и символы имеют значение.

2. Как Discord выбирает размер картинки

Большая картинка

Передаётся через LargeImage. Обычно это логотип, регион, город, подземелье или игровой мир.

Маленькая картинка

Передаётся через SmallImage. Обычно это бой, квест, карта, инвентарь, диалог или крафт.

В Portal нет отдельных типов «большая» и «маленькая». Роль определяется полем, в которое плагин передал Asset key.

Если большая картинка отсутствует в Portal, Discord может показать знак вопроса или визуально перенести маленькую картинку на основное место. Проверь строку Presence assets: large='...' small='...' в логе.

3. Базовая структура конфигурации

{
  "schema_version": 1,
  "discord": {
    "enabled": true,
    "application_id": "1527543892151373937"
  },
  "display": {
    "show_elapsed_time": true,
    "show_character_details": true,
    "show_location": true,
    "show_quest": true,
    "show_combat": true,
    "show_ui_state": true,
    "separator": " · "
  },
  "assets": {
    "large_image": "stb_logo",
    "large_text": "The Elder Scrolls V: Skyrim",
    "large_images": { },
    "small_images": { },
    "location_images": [ ]
  }
}

large_image — финальный резервный Asset key. Он используется, если подходящего правила локации или состояния нет.

4. Большие картинки глобальных состояний

"large_images": {
  "loading": "stb_logo",
  "main_menu": "stb_logo",
  "editing_character": "stb_logo",
  "playing": "stb_logo"
}
КлючКогда используется
loadingЭкран загрузки.
main_menuГлавное меню.
editing_characterСоздание или редактирование персонажа.
playingИгра, если правило конкретной локации не совпало.

5. Большая картинка под каждую локацию

Правила находятся в массиве assets.location_images и проверяются сверху вниз. Первое совпавшее правило побеждает.

Надёжный вариант: Editor ID

"location_images": [
  { "location": "WhiterunLocation", "image": "whiterun", "text": "Whiterun" },
  { "location": "SolitudeLocation", "image": "solitude", "text": "Solitude" },
  { "location": "WindhelmLocation", "image": "windhelm", "text": "Windhelm" },
  { "location": "RiftenLocation", "image": "riften", "text": "Riften" },
  { "location": "MarkarthLocation", "image": "markarth", "text": "Markarth" },
  { "worldspace": "DLC2SolstheimWorld", "image": "solstheim", "text": "Solstheim" },
  { "worldspace": "Blackreach", "image": "blackreach", "text": "Blackreach" },
  { "worldspace": "Tamriel", "image": "skyrim", "text": "Skyrim" }
]

Быстрый вариант: видимое название

{ "match": "Ривервуд", "image": "riverwood", "text": "Ривервуд" },
{ "match": "STB Dev Room", "image": "stb_room", "text": "STB Dev Room" }
СелекторНазначение
worldspaceEditor ID игрового мира.
locationEditor ID текущей локации или её родителя.
cellEditor ID конкретной ячейки.
matchЧасть отображаемого названия локации. Для ASCII регистр игнорируется.
imageAsset key большой картинки.
textТекст при наведении на большую картинку.
Порядок важен: конкретные города, помещения и подземелья размещай раньше общего правила Tamriel. Если указано несколько селекторов, совпасть должны все.

6. Маленькие иконки действий

"small_images": {
  "loading": "loading",
  "main_menu": "menu",
  "editing_character": "character",
  "exploring": "exploring",
  "quest": "quest",
  "combat": "combat",
  "menu": "menu",
  "map": "map",
  "inventory": "inventory",
  "dialogue": "dialogue",
  "crafting": "crafting",
  "waiting": "waiting"
}
СобытиеЧто охватываетAsset key
БойБой с известным или неизвестным противником.combat
КвестАктивная отображаемая задача.quest
КартаОткрытая карта мира или локации.map
ИнвентарьИнвентарь, магия, избранное, характеристики, торговля и контейнеры.inventory
ДиалогРазговор с NPC.dialogue
КрафтКузница, алхимия, зачарование и другие Crafting Menu.crafting
ОжиданиеСон или ожидание.waiting

Чтобы отключить конкретную иконку, укажи пустую строку: "map": "". Чтобы отключить все UI-состояния, установи "show_ui_state": false.

7. Приоритет событий

Бой → активный UI-контекст → активный квест → исследование

Пример: если во время квеста начинается бой, большая картинка локации остаётся прежней, а маленькая становится combat. После боя автоматически возвращается quest или exploring.

10. Диагностика

  1. Проверь, что application_id относится к тому же приложению, куда загружены assets.
  2. Открой публичный список: https://discord.com/api/v10/oauth2/applications/<APPLICATION_ID>/assets.
  3. Убедись, что нужный name появился в JSON ответа.
  4. Открой лог: %USERPROFILE%\Documents\My Games\Skyrim Special Edition\SKSE\DragonbornPresence.log.
  5. Найди строку вида: Presence assets: large='skyrim' small='combat'.
  6. После загрузки новых assets полностью перезапусти Discord и Skyrim для обновления кэша.
Знак вопроса или большая иконка боя: выбранный большой Asset key отсутствует или ещё не появился в API. Временно укажи существующий stb_logo, а после появления нового ключа верни нужное правило.

11. Что требует изменения C++

Новые локации и новые изображения для уже поддерживаемых событий добавляются только через JSON. Пересборка не нужна. Изменение C++ требуется лишь для совершенно нового игрового события, которого нет среди поддерживаемых режимов.

Документ относится к DragonbornPresence SE 2.4.0. Конфигурация обратно совместима со схемой schema_version: 1.
