DragonbornPresence SE — STB Presence

Настройка Discord Rich Presence

Две строки данных STB, большие изображения локаций и маленькие иконки загрузки и боя.

Русское практическое руководство для DragonbornPresence.json
Discord Game SDK — без OAuth и дополнительных окон авторизации

1. Что отображается

Во время игры плагин отправляет в Discord фиксированный снимок персонажа:

Поле DiscordФорматПример
detailsУровень: <уровень> • Камень: <камень>Уровень: 14 • Камень: Атронах
stateСмертей: <число> • Сложность: <режим>Смертей: 3 • Сложность: Приключение

Имя и раса персонажа, задания, меню, крафт, таймер, выбранные боги и другие состояния в Presence не передаются.

Обновление: данные STB, текущая локация, бой и имя цели опрашиваются на игровом потоке один раз в секунду. Перезапуск или пересборка для изменения значений не нужны.

Для значений STB необходимы установленный STB.esp и его MCM data-storage quest. Если отдельная форма недоступна, Discord Presence продолжает работать с резервным текстом.

3. Быстрый старт Discord

  1. Открой Discord Developer Portal и выбери приложение с ID из конфига.
  2. В разделе Rich Presence → Art Assets загрузи квадратные PNG.
  3. Проверь Asset keys: минимум stb_logo, loading, combat и нужные ключи локаций.
  4. Открой Data\SKSE\Plugins\DragonbornPresence.json.
  5. После изменения JSON полностью перезапусти Skyrim. Пересборка DLL не требуется.
Главное правило: имя файла PNG может быть любым, но Asset key в Discord Portal должен точно совпадать со строкой в JSON. Регистр и символы имеют значение.

5. Большая картинка текущей локации

В комплектном JSON находятся 414 правил. Они проверяются сверху вниз; первое полное совпадение выбирает большую картинку и текст подсказки.

"location_images": [
  { "match": "Коллегия Винтерхолда", "image": "winterholdcastle", "text": "Коллегия Винтерхолда" },
  { "location": "WhiterunLocation", "image": "whiteruncapital", "text": "Вайтран" },
  { "match": "Нилхейм", "image": "bandit", "text": "Нилхейм" }
]
Поле правилаНазначение
worldspaceEditor ID игрового мира.
locationEditor ID текущей локации или любого её родителя.
cellEditor ID текущей ячейки.
matchПодстрока отображаемого названия; для ASCII регистр не учитывается.
imageAsset key большой картинки.
textТекст при наведении; при пустом значении используется large_text.

Если указано несколько селекторов, должны совпасть все. Если ни одно правило не подошло, используется large_image. Новую локацию можно добавить только через JSON.

При смене противника подсказка обновляется следующим секундным опросом. После завершения боя маленькая иконка и сохранённое имя цели удаляются.

7. Ограничения интерфейса Discord

Discord сам рисует строки details и state. Клиент может показывать первую строку белой, вторую серой и добавлять перед строкой состояния значок людей/party status.

Это нельзя изменить из плагина: Discord Game SDK позволяет передать текст, timestamps, party data и assets, но не цвет шрифта и не оформление встроенных строк клиента. Плагин не отправляет party data; отображаемый значок относится к интерфейсу Discord.

8. Подготовка изображений

  • Большие картинки: квадратный PNG 1024×1024.
  • Маленькие иконки: прозрачный квадратный PNG 512×512 или 1024×1024.
  • Важные детали маленькой иконки размещай по центру: Discord обрезает её до круга.
  • Готовый набор локаций находится в DiscordAssets\markers-portal; имена файлов совпадают с Asset keys JSON.

9. Диагностика

  1. Проверь, что application_id относится к приложению с нужными assets.
  2. Открой https://discord.com/api/v10/oauth2/applications/<APPLICATION_ID>/assets и найди нужный name.
  3. Проверь лог %USERPROFILE%\Documents\My Games\Skyrim Special Edition\SKSE\DragonbornPresence.log.
  4. В строках обновления проверь large='...' и combat='...'.
  5. После загрузки новых assets полностью перезапусти Discord и Skyrim для обновления кэша.
