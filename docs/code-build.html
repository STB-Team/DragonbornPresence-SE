1. Строки 1–11: требования проекта

cmake_minimum_required(VERSION 4.2)
project(DragonbornPresence VERSION 3.1.8 LANGUAGES CXX)
set(CMAKE_CXX_STANDARD 23)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
include(FetchContent)
set(CMAKE_SKIP_INSTALL_RULES TRUE)
  • Минимальная версия проверяется до остальных commands.
  • project задаёт имя target namespace, версию и включает C++ toolchain.
  • Standard 23 нужен, в частности, для optional::transform.
  • REQUIRED ON запрещает compiler тихо откатиться на старый standard.
  • FetchContent загружает dependencies на configure stage.
  • Plugin не имеет cmake --install; skip защищает от install export rules транзитивных проектов.

2. Строки 13–64: зависимости

fmt 10.2.1

FetchContent_Declare фиксирует Git repository/tag, shallow clone и OVERRIDE_FIND_PACKAGE. Последнее перехватывает find_package(fmt) внутри CommonLib, чтобы использовалась та же загруженная версия.

spdlog 1.13.0

SPDLOG_FMT_EXTERNAL ON CACHE BOOL "" FORCE заставляет spdlog линковаться с внешним fmt вместо bundled copy. Это исключает две несовместимые копии formatter.

rapidcsv 8.83

DOWNLOAD_ONLY YES: library header-only, build target не нужен. Cache path устанавливается до CommonLib find_path. В собственных текущих src rapidcsv не вызывается, но dependency discovery CommonLib ожидает путь.

nlohmann/json 3.11.3

Header-only parser конфигурации и spell descriptions. FetchContent_MakeAvailable создаёт target nlohmann_json::nlohmann_json.

CommonLibSSE-NG 3.7.0

Tests выключаются cache variable. После загрузки вручную включается CommonLibSSE.cmake, потому что helper add_commonlibsse_plugin определён там.

Фиксированные tags: обновление dependency — отдельное изменение контракта. Нельзя просто заменить tag без полной сборки и запуска Skyrim.

3. Строки 66–94: Discord Game SDK

Условная загрузка

Build-local directory _discord_game_sdk. Если cpp/discord.h уже есть, download пропускается. Иначе file(DOWNLOAD) получает официальный ZIP 3.2.1 и возвращает status list.

Проверка status

list(GET ... 0 _discord_dl_err) берёт numeric error code. Ненулевой code вызывает message(FATAL_ERROR) с manual fallback path; configure прекращается.

Извлечение

file(ARCHIVE_EXTRACT) распаковывает SDK, временный ZIP удаляется.

C++ wrapper target

file(GLOB DISCORD_SDK_CPP ".../cpp/*.cpp")
add_library(discord_cpp STATIC ${DISCORD_SDK_CPP})
target_include_directories(discord_cpp PUBLIC ".../cpp")
target_link_libraries(discord_cpp PUBLIC ".../discord_game_sdk.dll.lib")

Wrapper sources компилируются в static library. Import library .dll.lib содержит linker stubs для runtime DLL. Wrapper принудительно C++17, как ожидает SDK; основной plugin остаётся C++23.

4. Строки 96–137: состав SKSE plugin target

Списки

DRAGONBORNPRESENCE_SOURCES содержит четыре .cpp; HEADERS — пять .h; RESOURCES — shipped JSON. Явный список лучше glob для собственного кода: добавление файла требует осознанного изменения build.

add_commonlibsse_plugin

  • Target и DLL name DragonbornPresence.
  • AUTHOR попадает в plugin metadata.
  • VERSION берётся из PROJECT_VERSION, заданной один раз.
  • USE_ADDRESS_LIBRARY включает runtime address resolution для разных Skyrim versions.

source_group

Только организация файлов в Visual Studio Solution Explorer. На binary не влияет.

Include directories

PRIVATE означает, что пути нужны при компиляции этого target, но не публикуются downstream targets.

Precompiled header

target_precompile_headers компилирует include/pch.h и автоматически подключает его в translation units согласно механизму CMake/MSVC.

5. Строки 138–140: линковка и compiler flags

target_link_libraries(DragonbornPresence PRIVATE
    discord_cpp delayimp nlohmann_json::nlohmann_json)
target_link_options(DragonbornPresence PRIVATE
    /DELAYLOAD:discord_game_sdk.dll)
target_compile_options(DragonbornPresence PRIVATE /utf-8)
  • discord_cpp — wrapper.
  • delayimp — MSVC runtime helper для delay loading.
  • nlohmann_json — include/interface target.
  • /DELAYLOAD не загружает SDK вместе с plugin DLL.
  • /utf-8 задаёт execution/source charset, необходимый русским literals и emoji.

6. Строки 142–151: оптимизированный Release с PDB

Блок исполняется только для MSVC:

  • MSVC_DEBUG_INFORMATION_FORMAT ProgramDatabase просит compiler генерировать symbol information.
  • /DEBUG:FULL создаёт полный PDB linker.
  • /OPT:REF удаляет неиспользуемые COMDAT/functions/data.
  • /OPT:ICF объединяет идентичные COMDAT blocks.

PDB не означает Debug configuration. Release optimization остаётся. Символы позволяют crash analyzer сопоставить address с именем/строкой, хотя optimizer может inline/reorder code.

7. Строки 153–209: staging, ZIP и MO2 deploy

Staging path

_package/SKSE/Plugins создаётся на configure. configure_file(... COPYONLY) копирует JSON без подстановки переменных.

POST_BUILD command

После успешного target build:

  1. SDK DLL копируется рядом с built plugin.
  2. Plugin DLL, target PDB и SDK DLL копируются в staging.
  3. JSON уже находится в staging.
  4. cmake -E tar cf --format=zip создаёт ZIP с четырьмя путями SKSE/Plugins/*.

Generator expressions $<TARGET_FILE:...> и $<TARGET_PDB_FILE:...> вычисляются для активной configuration и не hard-code Release path.

Optional deploy

Cache path DRAGONBORNPRESENCE_DEPLOY_DIR по умолчанию пуст. Если задан, второй POST_BUILD создаёт SKSE/Plugins в mod root и копирует DLL/PDB/SDK/JSON. copy_if_different избегает лишней записи одинаковых файлов. VERBATIM защищает аргументы с пробелами/скобками.

Практический эффект: локальная сборка перезаписывает JSON в MO2 исходной версией из config/. Runtime-правки только в папке MO2 потеряются при следующем build.

8. CMakePresets.json, все 41 строка

Schema

version: 6 — версия формата presets, не версия plugin. cmakeMinimumRequired дублирует executable requirement для preset consumer.

Configure preset vs2026

  • Generator: Visual Studio 18 2026.
  • Architecture: x64, обязательна SkyrimSE и Discord x86_64 SDK.
  • Binary directory: build/vs2026.
  • Cache deploy dir: локальный мод MO2.

Build presets

release и debug используют один configure preset, выбирают configuration и target. Multi-config Visual Studio generator хранит обе configurations в одном build tree.

cmake --preset vs2026
cmake --build --preset release

Loader semantics подробно разобраны в главе 2. Build system считает JSON resource/data, не компилирует его в DLL. Изменение применяется после полного перезапуска Skyrim, потому что config читается один раз в SKSEPluginLoad.

438 location records — экземпляры одной схемы. Их порядок является runtime priority. Indentation в текущем файле местами различается, но JSON whitespace не влияет на parser.

10. .github/workflows/release.yml, все 42 строки

Trigger и permissions

on:
  push:
    tags: ["v*"]
permissions:
  contents: write

Обычный push branch не создаёт release. Любой tag с prefix v запускает workflow. Write permission нужна для GitHub Release asset.

Job

Один job release на windows-2025-vs2026. Steps выполняются последовательно:

  1. actions/checkout@v4 получает tagged source.
  2. cmake --preset vs2026 configure.
  3. cmake --build --preset release build + package.
  4. PowerShell Test-Path явно проверяет ZIP, иначе throw.
  5. gh release create проверяет tag, генерирует notes, задаёт title и прикрепляет ZIP.

Expressions и folded scalar

${{ github.ref_name }} — имя tag. ${{ github.token }} — временный token run. YAML >- складывает много строк команды в одну, удаляя финальный newline.

GitHub Pages: данный workflow публикует Releases, не Pages. Pages использует статические файлы каталога docs/ согласно настройке репозитория.
