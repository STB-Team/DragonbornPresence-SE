Вход в плагин и связывание зависимостей

Два небольших translation unit отделяют ABI SKSE от application-кода: main.cpp принимает внешние lifecycle-сигналы, а DragonbornPresence.cpp владеет concrete adapters.

1. SKSEPluginLoad

src/main.cpp выполняет только bootstrap:

  1. создаёт файловый spdlog sink в каталоге SKSE log;
  2. инициализирует SKSE через SKSE::Init;
  3. загружает base JSON и пользовательский runtime overlay;
  4. вызывает DragonbornPresence::LoadConfig();
  5. регистрирует listener SKSE Messaging Interface;
  6. возвращает false при любом незавершённом обязательном шаге.

Функция обёрнута в try/catch. Исключение не пересекает ABI-границу SKSE, а Skyrim получает понятный отказ загрузки плагина.

3. Публичный фасад

include/DragonbornPresence.h экспортирует минимальный контракт:

void LoadConfig() noexcept;
void RegisterGameEventHandlers() noexcept;
void OnGameLoaded() noexcept;

Определения находятся в src/DragonbornPresence.cpp. Каждая функция добавляет собственный exception boundary и передаёт работу уже созданному coordinator или runtime adapter. RE/SKSE-типы в публичный заголовок не попадают.

4. Composition root

Конкретные объекты создаются в порядке зависимостей:

JsonConfigProvider
StbGameDataSource
DiscordPresenceClient
SkseLogger
PresenceCoordinator(config, gameData, presence, logger)
StbRuntimeAdapter(coordinator, logger)

PresenceCoordinator получает ссылки только на четыре application ports. Он не знает имён concrete classes. Замена внешнего транспорта или источника данных не требует менять application state machine.

5. Почему порядок объектов важен

C++ уничтожает объекты static storage в обратном порядке. StbRuntimeAdapter объявлен последним и поэтому уничтожается первым. Его std::jthread останавливается и присоединяется до уничтожения coordinator, logger, game data source и Discord client.

Инвариант: scheduler не должен пережить объекты, на которые он хранит non-owning references.

Не переносите concrete adapters в main.cpp и не меняйте порядок глобальных объектов без повторной проверки shutdown path.
