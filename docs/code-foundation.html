Чистые модели, правила и конфигурация

Core хранит язык приложения и детерминированные правила. JSON остаётся внешним форматом и преобразуется в core::Config отдельным adapter.

engine-independentowned dataUTF-8schema 1

1. Граница core

Core не включает RE, SKSE, Windows, Discord SDK, nlohmann/json или spdlog. Его файлы компилируются отдельной целью DragonbornPresenceCore и проверяются без игры.

external input → adapter → core model → pure rule → result

Если функция требует filesystem, registry, Skyrim singleton или Discord callback, она не относится к core.

3. UTF-8 utilities

src/core/TextUtils.cpp реализует три независимых правила:

  • IsValidUtf8 проверяет структуру последовательности и запрещённые Unicode scalar values;
  • LimitUtf8Bytes ограничивает Discord-строку по байтам без разрезания code point;
  • ContainsAsciiInsensitive выполняет case-insensitive поиск ASCII pattern без locale и временных lowercase-копий.

Преобразование CP1251 не относится к core: оно зависит от WinAPI и находится в STB adapter.

4. Сложность STB

DifficultyName(std::optional<int>) переводит raw Papyrus value в стабильную русскую подпись. Отсутствующее и неизвестное значение получает fallback не определена. Эта функция чистая: Papyrus lookup выполняет adapter, а core знает только число.

5. Выбор large asset

LocationAssetResolver обходит Config::locationImageRules сверху вниз. Для одного правила все непустые selectors объединяются через AND:

worldspace exact
AND cell exact
AND location occurs in parent hierarchy
AND displayName contains match

Первое полное совпадение завершает поиск. Если text правила пуст, используется глобальный largeText; если совпадений нет — largeImage и largeText.

Порядок JSON — поведение. Более узкие правила должны находиться раньше широких substring-правил.

6. JsonConfigProvider

Adapter объединяет shipped DragonbornPresence.json и пользовательский DragonbornPresence.user.json, затем отслеживает изменения обоих файлов.

  1. создаёт объект с безопасными defaults;
  2. парсит JSON и требует object root;
  3. проверяет schema version до изменения config;
  4. читает assets из base JSON, а toggle и layout строк — из user overlay;
  5. при hot reload возвращает только полностью разобранную замену, иначе сохраняет last-known-good;
  6. отбрасывает malformed location rules;
  7. сохраняет default для отсутствующего или неверного optional поля.

Adapter использует spdlog напрямую, а не SKSE logger. Поэтому JSON tests не требуют линковки CommonLibSSE.

7. schema_version

Поддерживаемая версия задаётся core::kSupportedConfigSchemaVersion = 1.

InputРезультат
1Документ загружается.
Поле отсутствуетLegacy-документ трактуется как schema 1; записывается warning.
Не integerВесь документ отклоняется, используются defaults.
Любая неподдерживаемая версияВесь документ отклоняется, используются defaults.

Отказ от частичного чтения future schema защищает приложение от интерпретации новых полей со старыми semantics.
