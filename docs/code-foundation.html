StoneRuntimeData и StbRuntimeData

StoneRuntimeData связывает невладеющий Skyrim pointer SpellItem* с владеющим готовым именем. StbRuntimeData кэширует два form pointers и vector камней. Плагин не удаляет Skyrim forms: владельцем остаётся игра.

CombatTargetData

Копирует имя и 16-bit level, не хранит actor pointer. Это важно: actor handle может стать недействительным, а простые значения безопасно переживают краткий разрыв target.

PlayerSnapshot

Value object одного чтения. optional<int> deaths различает настоящий ноль и отсутствие global. Строки принадлежат snapshot. inCombat=false задаёт безопасный default.

LargeAssetSelection

Два views на строки долгоживущего Config. Класс resolver хранит reference на config, а selection используется сразу в том же refresh. Если config заменить при живом selection, views стали бы invalid; текущий lifecycle не делает reload во время runtime.

ActivityPayload

Шесть views: details, state, large image/text, small image/text. Payload не владеет памятью и предназначен только для немедленного вызова client. Это экономит копии между coordinator и transport.

RefreshReason и ToLogLabel

enum class RefreshReason { kGameLoaded, kLoadingFinished, kCombat, kPoll };

enum class не смешивается неявно с int и требует квалификации. ToLogLabel — constexpr noexcept switch. Последний return "unknown" страхует невозможное/повреждённое значение.

Difficulty : int

Underlying type явно int, потому что Papyrus reader возвращает int. Значения 0–4 закреплены за контрактом STB. static_cast<Difficulty> не валидирует число, поэтому default в следующем switch обязателен.

4. namespace text

FromGameString, строки 185–189

if (!value || *value == '\0') return {};
return IsValidUtf8(value) ? std::string(value) : Cp1251ToUtf8(value);

Первая часть проверяет pointer до dereference благодаря short-circuit ||. return {} value-initializes пустой std::string. Ternary оставляет валидный UTF-8 неизменным, иначе предполагает CP1251.

LimitForDiscord, строки 192–202

  1. До 127 байт возвращается обычная копия.
  2. validSize начинает с byte offset 127.
  3. Маска (byte & 0xC0) == 0x80 распознаёт continuation byte UTF-8.
  4. Индекс движется назад до leading byte.
  5. substr(0, validSize) копируется во владеющую строку.

Проверка читает value[validSize]; она выполняется только когда исходная строка длиннее 127, поэтому индекс 127 существует.

ContainsAsciiInsensitive, строки 205–230

Пустой pattern всегда найден. Pattern длиннее value — никогда. Lambda foldAscii преобразует только A–Z. Внешний цикл выбирает offset, внутренний сравнивает все bytes. Complexity в худшем случае $O(nm)$; для коротких location names и rules это приемлемо.

Кириллица сравнивается byte-for-byte и чувствительна к регистру. Название функции честно говорит AsciiInsensitive, а не Unicode-insensitive.

FindObject(), строки 288–300

parent.find(key) не создаёт отсутствующий элемент. Iterator сравнивается с end(). &*value получает pointer на JSON value внутри parse tree; pointer используется только пока root живёт внутри Load().

ReadBool() и ReadString()

Оба принимают target по non-const reference. Target меняется только после проверки типа. Отсутствующий key молча оставляет default; присутствующий неверный type пишет warning. Это различает нормальную необязательность и ошибку пользователя.

ReadApplicationId()

  • Integer принимается только если положительный.
  • String копируется в encodedId, затем from_chars заполняет parsedId.
  • Structured binding const auto [end, error] раскладывает result.
  • Нужны одновременно: нет error, end достиг конца строки, ID > 0.
  • Строка "123x" отвергается, хотя числовой prefix parseable.

ReadLocationImageRules()

Array валидируется до цикла. reserve не меняет size, только capacity. Индекс нужен для точного warning path. Каждый элемент обязан быть object.

const auto readOptionalString = [&](const char* key, std::string& target) {
    // захватывает value, ruleIndex и isValid по reference
};

Lambda устраняет шесть одинаковых блоков. Capture [&] безопасен, потому что lambda не сохраняется и вызывается только внутри текущей iteration. Неверный тип ставит общий isValid=false.

hasSelector требует хотя бы одно условие. image обязательно. text необязателен. Валидное правило перемещается в vector. Только в самом конце весь vector перемещается в config.

schema_version: поле существует в shipped JSON, но ни одна строка ConfigLoader его не читает. Сейчас это человеческая метка, не runtime validation.

7. Семантика JSON-правил

{
  "worldspace": "...",  // optional exact Editor ID
  "location": "...",    // optional current/parent Location Editor ID
  "cell": "...",        // optional parent Cell Editor ID
  "match": "...",       // optional substring display name
  "image": "...",       // required Asset key
  "text": "..."         // optional hover text
}

Если указаны несколько selectors, действует логическое AND. Array order — priority. Loader не проверяет существование Editor IDs и Discord assets: C++ не имеет authoritative registry всех будущих runtime forms/assets.

В shipped JSON 438 entries. Это данные одного алгоритма, а не 438 разных веток C++. Документировать каждую запись как код бессмысленно; каждая имеет одинаковую схему и различается только значениями.

8. LocationAssetResolver, строки 438–507

Конструктор

explicit LocationAssetResolver(const model::Config& config) noexcept
    : config_(config) {}

explicit запрещает неявно превращать Config в resolver. Member initializer list инициализирует reference; reference нельзя оставить пустым и назначить позже.

Resolve()

При существующем player range-for перебирает const references на rules. Первое RuleMatches=true немедленно возвращается. Ternary выбирает rule text или config fallback. Без совпадения возвращается общий asset.

EditorIdEquals()

  • Пустой expected означает «selector не ограничивает правило» → true.
  • Null form при непустом expected → false.
  • GetFormEditorID() также может вернуть null.
  • expected == editorId сравнивает содержимое через string_view, не адреса pointers.

RuleMatches()

Каждый непустой selector может немедленно вернуть false. Location обходит linked parent chain:

for (auto* location = player->GetCurrentLocation(); location;
     location = location->parentLoc) {
    if (EditorIdEquals(location, rule.location)) { found = true; break; }
}

Цикл завершится на null parent. После structural selectors последним проверяется display-name match. Short-circuit rule.match.empty() || ... не запускает поиск для пустого pattern.
