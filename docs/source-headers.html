Каталоги заголовков повторяют слои архитектуры. Include path сразу показывает направление зависимости.

Корневой фасад

include/DragonbornPresence.h — три функции lifecycle без RE/SKSE includes. include/pch.h — precompiled header только production plugin target.

core/

HeaderНазначение
Config.hDefaults и configuration model.
PlayerSnapshot.hOwned gameplay snapshot.
LocationContext.h, LocationAssets.hLocation input, rules и result.
PresencePayload.hTransport-neutral outgoing data.
RefreshReason.hRefresh enum и log labels.
Difficulty.h, TextUtils.h, LocationAssetResolver.hЧистые правила.

application/

PresenceCoordinator.h объявляет use-case state machine. Подкаталог application/ports содержит четыре интерфейса внешнего мира. Ни один header application не включает RE/SKSE или Discord SDK.

adapters/SkyrimTrueBeliever/

  • StbGameDataSource.h — concrete game data port и RE-owned cache;
  • StbRuntimeAdapter.h — event sinks, scheduler и task state;
  • PapyrusValueReader.h — header-only constrained templates;
  • StbStringEncoding.h — CP1251 conversion;
  • SkseLogger.h — concrete logger port.

adapters/discord/ и adapters/config/

  • DiscordSdkLoader.h — Windows bootstrap functions;
  • DiscordPresenceClient.h — concrete Presence port и SDK ownership;
  • JsonConfigProvider.h — concrete config port.
