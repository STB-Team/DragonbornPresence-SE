Сборка DLL не перезаписывает рабочий JSON в MO2. Изменение JSON применяется только после полного перезапуска Skyrim.

Правила локаций

"location_images": [
  { "worldspace": "DLC2SolstheimWorld", "image": "solstheim", "text": "Солстхейм" },
  { "location": "WhiterunLocation", "image": "whiteruncapital", "text": "Вайтран" },
  { "match": "Нилхейм", "image": "bandit", "text": "Нилхейм" }
]
ПолеУсловие
worldspaceEditor ID игрового мира
locationEditor ID локации или любого родителя
cellEditor ID текущей ячейки
matchподстрока имени; ASCII без учёта регистра
imageобязательный Asset key
textподсказка или fallback large_text

Проверка идёт сверху вниз; первое полное совпадение останавливает поиск. В текущем конфиге 438 правил.

Сборка и автокопирование

Visual Studio 2026, CMake 4.2+, генератор Visual Studio 18 2026, x64, toolset v145.

cmake --preset vs2026 -DDRAGONBORNPRESENCE_DEPLOY_DIR="D:/Stb/[STB] Mod Organizer/mods/DragonbornPresence SE"
cmake --build --preset release

После сборки DLL и Discord SDK обновляются в <корень мода>\SKSE\Plugins. JSON сохраняется.

ZIP:
D:\Dev\STB-Discord-Integration\DragonbornPresence-SE\build\vs2026\DragonbornPresence.zip

Архив содержит корневой каталог SKSE и устанавливается напрямую через MO2 без FOMOD.

Диагностика

%USERPROFILE%\Documents\My Games\Skyrim Special Edition\SKSE\DragonbornPresence.log

Перед инициализацией проверяются обработчик discord://, исполняемый файл и соответствующий процесс. Если Discord Desktop не установлен или не запущен, точка входа SKSE возвращает отказ до инициализации плагина. Если клиент завершится позже либо SDK вернёт ошибку, callback-loop и вся дальнейшая работа Presence немедленно прекращаются без повторных вызовов SDK.

Таймер

Discord Game SDK initialized for application ...; session_start=1784380000.
Presence updated; session_start=1784380000.
Presence updated; session_start=1784380000.

Все значения session_start в рамках одного запуска должны совпадать.

Локация

[poll] level=... deaths=... location='...' large='...' combat='...'

Строка снимка записывается только при изменении Presence. Неизменный секундный polling не создаёт повторных записей.
