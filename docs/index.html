Установка

Менеджер модов

Установите Release ZIP как обычный мод. В архиве уже есть FOMOD и структура SKSE.

Вручную

Скопируйте каталог SKSE из архива в каталог Data игры.

Data\SKSE\Plugins\DragonbornPresence.dll
Data\SKSE\Plugins\discord_game_sdk.dll
Data\SKSE\Plugins\DragonbornPresence.json

Требования

  • Skyrim SE или AE, соответствующий SKSE64 и Address Library;
  • Skyrim True Believer с STB.esp;
  • Discord Desktop с включённой игровой активностью.

Настройка Discord

  1. Создайте приложение в Discord Developer Portal.
  2. Скопируйте Application ID в discord.application_id.
  3. Откройте Rich Presence → Art Assets и загрузите квадратные PNG.
  4. Добавьте ключи stb_logo, loading, combat и ключи используемых локаций.
  5. Полностью перезапустите Discord и Skyrim.
Asset key чувствителен к регистру и должен точно совпадать со строкой в JSON. Имя PNG-файла значения не имеет.

Конфигурация JSON

Рабочий файл: Data\SKSE\Plugins\DragonbornPresence.json.

{
  "schema_version": 1,
  "discord": {
    "enabled": true,
    "application_id": "1527543892151373937"
  },
  "assets": {
    "large_image": "stb_logo",
    "large_text": "Skyrim True Believer",
    "small_images": {
      "loading": "loading",
      "combat": "combat"
    },
    "location_images": [
      {
        "location": "WhiterunLocation",
        "image": "whiteruncapital",
        "text": "Вайтран"
      }
    ]
  }
}
КлючНазначение
discord.enabledвключает или отключает Presence
discord.application_idID приложения Discord
assets.large_imageрезервная большая картинка
assets.large_textрезервная подсказка картинки
assets.small_images.loadingиконка загрузки
assets.small_images.combatиконка боя

Правила локаций

Правила проверяются сверху вниз. Первое полное совпадение выбирает картинку и подсказку.

"location_images": [
  { "worldspace": "DLC2SolstheimWorld", "image": "solstheim", "text": "Солстхейм" },
  { "location": "WhiterunLocation", "image": "whiteruncapital", "text": "Вайтран" },
  { "match": "Нилхейм", "image": "bandit", "text": "Нилхейм" }
]
ПолеУсловие
worldspaceEditor ID игрового мира
locationEditor ID текущей локации или её родителя
cellEditor ID текущей ячейки
matchподстрока отображаемого названия
imageобязательный Asset key
textподсказка; пустая заменяется large_text

Если селекторов несколько, должны совпасть все. Если правил нет — используется large_image. В комплекте находятся 414 правил.

Поведение во время игры

Загрузка

Статус Загрузка, большая резервная картинка и маленькая иконка loading.

Бой

Иконка combat; подсказка В бою с <цель> или В бою.

Обычная игра

Маленькая иконка отсутствует. Большая картинка следует правилам локации.

Discord недоступен

Плагин продолжает работать; ошибка записывается в лог без остановки Skyrim.

Таймер сессии: начинается при загрузке плагина и использует один timestamp до закрытия Skyrim. Смена локации, загрузка и бой его не сбрасывают.

Каждое текстовое поле ограничивается 127 байтами по границе UTF-8. Одинаковые payload повторно не отправляются.

Диагностика

%USERPROFILE%\Documents\My Games\Skyrim Special Edition\SKSE\DragonbornPresence.log
  1. Проверьте наличие DLL, Discord SDK и JSON в Data\SKSE\Plugins.
  2. Сверьте Application ID и Asset keys с Discord Developer Portal.
  3. Найдите в логе сообщения Config:, Presence updated и large='...'.
  4. После изменения JSON или Assets полностью перезапустите Discord и Skyrim.

Сборка Visual Studio 2026

Нужны Visual Studio 2026 с компонентом разработки C++ и CMake 4.2 или новее. Используются генератор Visual Studio 18 2026, архитектура x64 и toolset v145.

git clone https://github.com/STB-Team/DragonbornPresence-SE.git
cd DragonbornPresence-SE
cmake --preset vs2026
cmake --build --preset release

Архив: build\vs2026\DragonbornPresence.zip.

Debug

cmake --build --preset debug

Автокопирование в MO2

cmake --preset vs2026 -DDRAGONBORNPRESENCE_DEPLOY_DIR="D:/Mod Organizer/mods/DragonbornPresence SE/SKSE/Plugins"
cmake --build --preset release

GitHub Release с готовым ZIP

Workflow запускается по новому тегу v* на runner windows-2025-vs2026.

git tag -a vX.Y.Z -m "DragonbornPresence X.Y.Z"
git push origin vX.Y.Z

Замените vX.Y.Z новым номером версии. GitHub Actions соберёт Release, создаст страницу релиза и прикрепит DragonbornPresence.zip.

Workflow запрашивает только contents: write. Разрешение задано непосредственно в YAML и не требует переключения глобального значения по умолчанию.
