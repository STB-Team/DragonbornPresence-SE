STB

Текущий STB.esp и его MCM data-storage quest.

Discord

Установленный и уже запущенный Discord Desktop, Application ID 1527543892151373937 и assets команды. Без активного процесса инициализация плагина прекращается до загрузки discord_game_sdk.dll; Discord автоматически не запускается.

Обновление

Один ограниченный callback и снимок игрового состояния раз в 500 мс на game thread.

Рабочий JSON

<корень мода MO2>\SKSE\Plugins\DragonbornPresence.json

{
  "schema_version": 1,
  "discord": {
    "enabled": true,
    "application_id": "1527543892151373937"
  },
  "assets": {
    "large_image": "stb_logo",
    "large_text": "Skyrim True Believer",
    "small_images": {
      "loading": "loading",
      "combat": "combat"
    },
    "location_images": []
  }
}

schema_version проверяется до остальных полей. Версия 1 поддерживается; отсутствующее поле считается legacy schema 1, а неверная или неподдерживаемая версия возвращает полный default config.

JSON копируется в MO2 только локальным user preset с настроенным deploy-каталогом. Изменение JSON применяется игрой после полного перезапуска Skyrim.

Правила локаций

"location_images": [
  { "worldspace": "DLC2SolstheimWorld", "image": "solstheim", "text": "Солстхейм" },
  { "location": "WhiterunLocation", "image": "whiteruncapital", "text": "Вайтран" },
  { "match": "Нилхейм", "image": "bandit", "text": "Нилхейм" }
]
ПолеУсловие
worldspaceEditor ID игрового мира
locationEditor ID локации или любого родителя
cellEditor ID текущей ячейки
matchподстрока имени; ASCII без учёта регистра
imageобязательный Asset key
textподсказка или fallback large_text

Проверка идёт сверху вниз; первое полное совпадение останавливает поиск. В текущем конфиге 439 правил.

Сборка и локальное развёртывание

Visual Studio 2026, CMake 4.2+, генератор Visual Studio 18 2026, x64, toolset v145.

cmake --preset vs2026
cmake --build --preset release

Общий preset переносим и не содержит MO2 path. Локальный путь задаётся в ignored CMakeUserPresets.json; presets vs2026-local и release-local развёртывают runtime-файлы в выбранный мод.

ZIP:
build/vs2026/DragonbornPresence.zip

Архив содержит корневой каталог SKSE и устанавливается напрямую через MO2 без FOMOD.

Автоматические тесты и CI

cmake --preset vs2026-tests
cmake --build --preset tests
ctest --preset tests

Три CTest-цели проверяют core, application, JSON schema и кодировки. Отдельный CI job собирает production SKSE-плагин и запускает строгую проверку ZIP manifest.

Тесты не заменяют runtime smoke test: после изменения RE/SKSE adapter нужна Release-сборка и проверка через SKSE в текущей сборке STB.

Диагностика

%USERPROFILE%\Documents\My Games\Skyrim Special Edition\SKSE\DragonbornPresence.log

Перед инициализацией проверяются обработчик discord://, исполняемый файл, соответствующий процесс и локальный Discord SDK. Плагин ничего не устанавливает и не запускает. Ошибка клиента, SDK, callback или C++-исключение переводит интеграцию в окончательно остановленное состояние; Skyrim продолжает работу.

Таймер

Discord Game SDK initialized for application ...; session_start=1784380000.
Presence updated; session_start=1784380000.
Presence updated; session_start=1784380000.

Все значения session_start в рамках одного запуска должны совпадать.

Локация

[poll] level=... deaths=... location='...' large='...' combat='...'

Строка снимка записывается только при изменении Presence. Неизменный polling раз в 500 мс не создаёт повторных записей.

Релиз 4.0.0

git tag -a v4.0.0 -m "DragonbornPresence 4.0.0"
git push origin v4.0.0

Workflow проверяет соответствие тега CMake-версии, запускает все тесты, собирает Release, проверяет точный ZIP manifest и только затем публикует архив.

Для следующего релиза сначала обновить версию в CMakeLists.txt, затем создать новый тег.
