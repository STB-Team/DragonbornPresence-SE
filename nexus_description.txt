[font=Tahoma][size=5][b]DRAGONBORN PRESENCE[/b][/size]
[size=3]DISCORD RICH PRESENCE SKSE PLUGIN[/size][/font][/center]

[font=Tahoma]Dragonborn Presence publishes the Skyrim True Believer character snapshot to Discord: level and active standing stone on the first line, deaths and selected difficulty on the second. The large image follows the current location; loading and combat use dedicated small icons.[/font]

[font=Tahoma][url=https://www.nexusmods.com/skyrim/mods/97371]Legendary Edition version[/url]   [url=https://github.com/STB-Team/DragonbornPresence-SE]Source Code[/url][/font]

[font=Tahoma][size=4][b]— What it shows —[/b][/size][/font]

[font=Tahoma][list]
[*][b]Character[/b] — [i]Уровень: 14 • Камень: Атронах[/i].
[*][b]STB progress[/b] — [i]Смертей: 3 • Сложность: Приключение[/i].
[*][b]Location art[/b] — 414 ordered rules select city, settlement, dungeon, and encounter images.
[*][b]Combat[/b] — the small [font=Courier New]combat[/font] icon appears during combat; hover text shows [i]В бою с <имя противника>[/i] when Skyrim exposes the target.
[*][b]Loading[/b] — the small [font=Courier New]loading[/font] icon replaces stale gameplay status during loading screens.
[*][b]Updates[/b] — STB values, combat, and location art refresh once per second without per-frame polling.
[/list][/font]

[font=Tahoma][size=4][b]— How to install —[/b][/size][/font]

[font=Tahoma][b]Mod manager (recommended):[/b] Install the archive normally. The FOMOD installer has no language-selection step.

[b]Manual:[/b] Extract the archive and copy [font=Courier New]SKSE\[/font] into your Skyrim [font=Courier New]Data\[/font] directory.

No .esp. No plugins to enable. Just drop and play.

[b]Upgrading from an old version:[/b] If you have [font=Courier New]discord_game_sdk.dll[/font] sitting in the root of your Skyrim directory (next to [font=Courier New]SkyrimSE.exe[/font]), delete it — it was placed there by older versions and is no longer needed. The file now lives in [font=Courier New]Data\SKSE\Plugins\[/font] where it belongs.[/font]

[font=Tahoma][size=4][b]— Language —[/b][/size][/font]

[font=Tahoma]All Presence state labels are built into the plugin in Russian. There is no language selector or external locale file.[/font]

[font=Tahoma][size=4][b]— Compatibility —[/b][/size][/font]

[font=Tahoma]Pure C++ SKSE plugin with no plugin-owned .esp or Papyrus scripts. STB character values require the Skyrim True Believer setup and [font=Courier New]STB.esp[/font]. Missing STB values degrade to safe fallback text instead of stopping the plugin.[/font]

[font=Tahoma][size=4][b]— Version 2.0 —[/b][/size][/font]

[font=Tahoma]Version 2.0 is a full rewrite. If you used version 1.x, here's what changed:[/font]

[font=Tahoma][b]No more .esp[/b]
The old version shipped a Papyrus script and a plugin file to track the player's location. Version 2.0 uses native C++ SKSE event hooks ([font=Courier New]TESActorLocationChangeEvent[/font], [font=Courier New]TESCellFullyLoadedEvent[/font]) — the .esp is gone entirely. Fewer files, no load order concerns, no script overhead.

[b]Any runtime[/b]
Version 1.x only worked on a specific Skyrim runtime (1.5.97). Version 2.0 uses the address library and runs on any version of Skyrim SE or AE without changes.

[b]Discord Game SDK instead of discord-rpc[/b]
Replaced the archived [font=Courier New]discord-rpc[/font] library with the current Discord Game SDK. More reliable connection, proper async callbacks, no stale presence.

[b]Better location display[/b]
The old version could show an empty location in areas without a named location record. Version 2.0 traverses the full location hierarchy to find the nearest named parent (e.g. a room inside a dungeon will show the dungeon name), and falls back to the last known location when crossing boundaries.

[b]Correct character name after creation[/b]
The old version sometimes showed the default name ("Prisoner") for a few seconds after character creation. Version 2.0 reads the actor's display name directly and defers the presence update until Skyrim has committed the new name.[/font]
